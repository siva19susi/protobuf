OTDR

Method NameRequest TypeResponse TypeDescription
Initiate InitiateRequest InitiateResponse stream

Initiate triggers an optical time domain reflectometer (OTDR) trace to be run on a specified port. The results of the trace may be sent back in the response or saved on the device to be retrieved later. If the information is saved on the device a path to the file must be returned which can later be used by File.Get or File.TransferToRemote. The RPC should initiate the OTDR trace and wait until the OTDR trace has completed and the device has processed the results before returning. If the RPC is cancelled while in operation, the running OTDR trace should stop.

InitiateRequest

Field Type Label Description
component gnoi.types.Path

Absolute path to the /components/component list member in the OpenConfig model corresponding to the port to perform the OTDR trace from. (e.g., /components/component[name=otdr-1-1])

results_method InitiateRequest.ResultsMethod repeated

Describes how the results of the OTDR trace should be made available. More than one option may be specified if desired.

configuration OTDRConfiguration

Describes the configuration used to perform an OTDR trace.

label string

If specified, the device will label the OTDR trace so it can be recalled later. For example, the initial OTDR trace measured during span commissioning might be labeled "baseline".

InitiateResponse

IntitiateResponse describes the possible response messages that will be streamed after the RPC is initiated. While the OTDR is running, "progress" messages should be sent at a regular interval. When the OTDR trace is completed, a final "results" or "error" message should be streamed before the RPC is terminated.

OTDRConfiguration

OTDR configuration options which can be set on the device. It is not required that every field be set as some fields may not apply for a given implementation.

Field Type Label Description
acquisition_time_s uint32

The time in seconds in which the OTDR trace will run continuously to collect data.

pulse_width_ns float

The pulse width in nanoseconds which defines how long pulses sent by the OTDR will be during the test.

wavelength_mhz uint64

The wavelength in MHz that will be sent by the OTDR. This may be left blank if the OTDR only supports one wavelength.

range_m float

The maximum fiber distance range in meters that the OTDR trace will support.

fiber_type FiberTypeProfile

The type of fiber that is being measured.

sampling_resolution_m float

The sampling resolution in meters.

InitiateError.Type

NameNumberDescription
UNSPECIFIED 0

An unspecified error. Must use the detail value to describe the issue.
ALREADY_IN_PROGRESS 1

An existing OTDR trace is already in progress.
HARDWARE_FAILURE 2

The OTDR hardware has failed and cannot complete an OTDR trace.

InitiateProgress.State

The State enum defines possible states that the OTDR trace may be in while running the operation.

NameNumberDescription
UNKNOWN 0

RUNNING 1

The OTDR is currently performing an OTDR trace.
PENDING 2

The OTDR is pending underlying resources being available.
COMPLETE 3

The OTDR trace was completed.
