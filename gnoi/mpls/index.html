MPLSPingRSVPTEDestination

MPLSPingRSVPTEDestination specifies the destination for an MPLS Ping in terms of an absolute specification of an RSVP-TE LSP. It can be used to identify an individual RSVP-TE session via which a ping should be sent.

Field Type Label Description
src string

The IPv4 or IPv6 address used by the system initiating (acting as the head-end) of the RSVP-TE LSP.

dst string

The IPv4 or IPv6 address used by the system terminating (acting as the tail-end) of the RSVP-TE LSP.

extended_tunnel_id uint32

The extended tunnel ID of the RSVP-TE LSP, expressed as an unsigned, 32b integer.

MPLSPingRequest

MPLSPingRequest specifies the parameters that should be used as input from the client, to a system that is initiating an RFC4379 MPLS ping request.

Field Type Label Description
ldp_fec string

The LDP forwarding equivalence class that the ping should be sent to expressed as an IPv4 or IPv6 prefix.

fec129_pwe MPLSPingPWEDestination

The FEC129 PWE to which the LSP ping should be sent.

rsvpte_lsp_name string

The name of an RSVP-TE LSP via which the ping should be sent. The system should locally resolve the name to a particular RSVP-TE session.

rsvpte_lsp MPLSPingRSVPTEDestination

An exact specification of an RSVP-TE LSP to which the system should send an MPLS ping message. TODO(robjs): L3VPN, BGP-LU destination types. See RFC4379.

reply_mode MPLSPingRequest.ReplyMode

How the target LER should respond to the LSP ping.

count uint32

The number of MPLS echo request packets to send. default=3

size uint32

The size (in bytes) of each MPLS echo request packet. default=64

source_address string

The source IPv4 address that should be used in the request packet.

mpls_ttl uint32

The MPLS TTL that should be set in the packets sent.

traffic_class uint32

The value of the traffic class (TC, formerly known as EXP) bits that should be set in the MPLS ping packets.

MPLSPingResponse

MPLSPingResponse (MPLSPong?) is sent from the target to the client based on each MPLS Echo Response packet it receives associated with an input MPLSPing RPC.

Field Type Label Description
seq uint32

The sequence number of the MPLS Echo Reply.

response MPLSPingResponse.EchoResponseCode

The response that was received from the egress LER.

response_time uint64

The time (in nanoseconds) between transmission of the Echo Response, and the echo reply.

ClearLSPRequest.Mode

NameNumberDescription
DEFAULT 0

Same as NONAGGRESSIVE.
NONAGGRESSIVE 0

Reoptimize the LSP using the current bandwidth.
AGGRESSIVE 1

Reoptimize the LSP using the current bandwidth. Only use IGP metric in calcuation.
RESET 2

Reset and restart all LSPs that originated from this routing device.
AUTOBW_AGGRESSIVE 3

Apply the highest bandwidth collected on a tunnel without waiting for the current application period to end. Only use IGP metric in calcuation.
AUTOBW_NONAGGRESSIVE 4

Apply the highest bandwidth collected on a tunnel without waiting for the current application period to end.

MPLSPingRequest.ReplyMode

NameNumberDescription
IPV4 0

Reply via an IPv4 packet to this system.
ROUTER_ALERT 1

Reply with a labeled packet with the router alert bit set.

MPLSPingResponse.EchoResponseCode

NameNumberDescription
SUCCESS 0

A successful echo response was received.
NOT_SENT 1

The MPLS ping packet was not sent, for an unknown reason.
TIMEOUT 2

The local system timed out waiting for an LSP ping response. TODO(robjs): Add additional error codes.
