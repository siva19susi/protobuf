AuthorizationPolicy

AuthorizationPolicy includes rules and user/group information. The service which updates the policies will provide both a created_on - int64 - seconds since Jan 01 1970 00:00:00 version - string - unique among the implementation's policies deployed.

Field Type Label Description
rules AuthorizationRule repeated

groups Group repeated

Groups of usernames collected for simplicity of policy expression.

AuthorizationRule

Field Type Label Description
id string

Policy Identifier, a unique string per policy/rule, eg: uuid.

user string

group string

path gnmi.Path

Path is the request path, longest prefix (by path elements from left to right).

action Action

Permit or deny the user/group access to the path specified.

mode Mode

Permit Read or Write access to the path specified.

Group

Group is a group of users, groups are a construct of the policy configuration as a method to collect many users with the same authorizations together.

Field Type Label Description
name string

users User repeated

User

User is a singular username used only in the matching criteria.

Action

Action is the defined action for an AuthorizationPolicy.

NameNumberDescription
ACTION_UNSPECIFIED 0

ACTION_DENY 1

ACTION_PERMIT 2

Mode

Mode is: READ: Ability to read/subscribe to data from the model. WRITE: Ability to mutate/write updates to data in the model.

NameNumberDescription
MODE_UNSPECIFIED 0

MODE_READ 1

MODE_WRITE 2
