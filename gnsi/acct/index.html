AuthDetail

Details of authentication - for gRPC or shell/vendor-CLI.

Field Type Label Description
identity string

Identity string is used to identify the user that executed the command. For instance, it could be spiffe-id in the case of gRPC or unix-style user-name in the case of shell/vendor-CLI.

privilege_level uint32

Privilege level configured on the system.

status AuthDetail.AuthenStatus

deny_cause string

In case of STATUS_DENY, cause for the deny

Arguments to a command above.

GrpcService

Command details for openconfig gNxI commands.

Field Type Label Description
service_type GrpcService.GrpcServiceType

rpc_name string

rpc_name is the rpc to which the request was sent, examples: gnmi.Set() gnoi.os.Install() gnsi.cert.Rotate()

payloads google.protobuf.Any repeated

A string of bytes that are gRPC encoded. Best to decode using PROTO_ANY.

config_metadata string

config_metadata can be used to track all set requests that were to be applied atomically.

Record

An accounting record message is generated everytime the user types a shell/vendorCLI command or sends a gNxI setRequest.

Session information

The time when the system recorded this event. Recorded as nanoseconds since epoch.

failure_cause string

In case of STATUS_FAILURE, cause for the failure

Authentication related details

task_ids string repeated

Optional repeated task_id that represent tasks that were used to accomplish the request on the system.
